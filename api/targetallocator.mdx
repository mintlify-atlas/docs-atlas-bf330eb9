## Overview

The `TargetAllocator` is a standalone custom resource for distributing Prometheus scrape targets across a fleet of OpenTelemetry Collectors. It can be used independently of the `OpenTelemetryCollector` CR. An embedded version is also available inside `OpenTelemetryCollector.spec.targetAllocator`.

| Field | Value |
|-------|-------|
| **API Group** | `opentelemetry.io` |
| **API Version** | `v1alpha1` |
| **Kind** | `TargetAllocator` |
| **Scope** | Namespaced |

## Example

```yaml
apiVersion: opentelemetry.io/v1alpha1
kind: TargetAllocator
metadata:
  name: my-target-allocator
  namespace: observability
spec:
  managementState: managed
  replicas: 1
  image: ghcr.io/open-telemetry/opentelemetry-operator/target-allocator:latest
  serviceAccount: target-allocator
  allocationStrategy: consistent-hashing
  filterStrategy: relabel-config
  collectorNotReadyGracePeriod: 30s
  resources:
    requests:
      cpu: 100m
      memory: 128Mi
    limits:
      cpu: 500m
      memory: 512Mi
  env:
    - name: MY_POD_NAMESPACE
      valueFrom:
        fieldRef:
          fieldPath: metadata.namespace
  tolerations:
    - key: "node-role"
      operator: "Equal"
      value: "observability"
      effect: "NoSchedule"
  observability:
    metrics:
      enableMetrics: true
  podDisruptionBudget:
    maxUnavailable: 1
  global:
    scrape_interval: 60s
    evaluation_interval: 60s
  scrapeConfigs:
    - job_name: static-example
      static_configs:
        - targets:
            - "my-service:9090"
  prometheusCR:
    enabled: true
    scrapeInterval: 30s
    serviceMonitorSelector:
      matchLabels:
        monitoring: enabled
    podMonitorSelector:
      matchLabels:
        monitoring: enabled
    serviceMonitorNamespaceSelector: {}
    podMonitorNamespaceSelector: {}
    allowNamespaces:
      - observability
      - production
  livenessProbe:
    initialDelaySeconds: 5
    timeoutSeconds: 5
    periodSeconds: 10
    failureThreshold: 3
  readinessProbe:
    initialDelaySeconds: 5
    timeoutSeconds: 3
    periodSeconds: 10
```

## Spec Fields

### Core Fields

<ParamField path="spec.managementState" type="string" default="managed">
  Controls whether the operator reconciles this resource. One of `managed` or `unmanaged`.
</ParamField>

<ParamField path="spec.replicas" type="integer" default="1">
  Number of Target Allocator pod instances. Values greater than `1` only make sense with the `consistent-hashing` allocation strategy.
</ParamField>

<ParamField path="spec.image" type="string">
  Container image for the Target Allocator. If not set, the operator uses its bundled default.
</ParamField>

<ParamField path="spec.imagePullPolicy" type="string">
  Image pull policy. One of `Always`, `Never`, or `IfNotPresent`.
</ParamField>

<ParamField path="spec.serviceAccount" type="string">
  Name of an existing `ServiceAccount`. When set, the operator will not create one automatically.
</ParamField>

<ParamField path="spec.allocationStrategy" type="string" default="consistent-hashing">
  Strategy for distributing targets among collectors.

  | Value | Description |
  |-------|-------------|
  | `consistent-hashing` | Targets are distributed consistently — supports high availability with multiple replicas. |
  | `least-weighted` | Targets are assigned to the collector with the fewest currently assigned targets. |
  | `per-node` | Each target is assigned to the collector running on the same node. Use only with `daemonset` mode collectors. **Note**: ignores targets without a node (e.g., control plane components). |
</ParamField>

<ParamField path="spec.filterStrategy" type="string" default="relabel-config">
  Strategy for filtering targets before allocation. Currently only `relabel-config` is supported (drops targets based on Prometheus `relabel_config`). An empty string disables filtering.
</ParamField>

<ParamField path="spec.collectorNotReadyGracePeriod" type="string" default="30s">
  Duration the Target Allocator waits before reassigning targets from a not-Ready collector. This avoids expensive reallocations during transient failures. Must be a valid Go duration string (e.g., `"30s"`, `"1m"`).
</ParamField>
